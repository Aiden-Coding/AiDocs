# Workspace Rules

- Markdown 写入/修改规范：所有 Markdown 文件必须符合 Markdownlint 规则。具体要求如下：
  1. **标题无尾部标点**：所有标题（如 `#` 至 `######`）末尾绝对不能有标点符号，包括中文冒号 `：`、英文冒号 `:`、问号 `？` 或 `?` 等。
  2. **空行控制**：代码块和列表前后必须留有且仅有**一个**空行。禁止连续出现两个或多个空行，文件末尾也不得留有连续空行。
  3. **MDX 兼容与转义**：泛型尖括号 `< >` 与公式或哈希 Tag 中的大括号 `{ }` 必须使用反引号或行内代码块包裹，防止解析崩溃。
- 中英文排版与拼写纯净度：
  1. 严禁在生成或替换中文内容时混入低级英文词汇（如 `of`、`and`、`the`、`in` 等代替中文的“的”、“和”、“在”）。
  2. 每次完成 `replace_file_content` 或代码生成后，必须对生成的中文文本进行二次复查，确保中英文符号使用正确且无错乱。
